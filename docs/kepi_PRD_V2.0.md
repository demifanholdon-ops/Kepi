 ## 1. 需求概述
 
 ### 1.1 为什么需要 V2.0
 
 V1.6 是一套"压缩版标准自走棋"——它砍了关卡数（10→6）、削了数值，但保留了经典自走棋的完整经济骨架（利息、连胜连败、3 星追求、后勤无敌位）。这套骨架天然导向"攒钱→吃利息→滚雪球"的慢节奏体验，与黑客松路演所需的"10 分钟情感冲击"存在根本矛盾。
 
 V2.0 不修修补补，而是**从底层心流出发重新设计**：一局 10 分钟、6 回合、每回合都是生存抉择。删除所有"安全理财"机制，把资源逼到极限，让玩家在"保命"与"保信"之间反复撕裂。
 
 ### 1.2 重构核心思路
 
 > "不是压缩一个大游戏，而是设计一个小游戏。"
 
 | 维度 | V1.6 思路 | V2.0 思路 |
 |---|---|---|
 | 时长心流 | 40 分钟大局的压缩版 | 10 分钟原生微型局 |
 | 经济模型 | 标准自走棋（利息+连胜+3星） | 极限生存（固定底薪+典当抉择） |
 | 后勤角色 | 无敌挂件（不占人口、不上场） | 上场棋子（可被杀、需保护） |
 | 家园修复 | 纯视觉升级 | 视觉升级 + 战斗 Buff |
 | 核心张力 | 理财 vs 战力 | 护信 vs 保命 |
 | AI 落点 | 数字客批生成 + 对手难度 | 聚焦机制叙事生成（动态侨批旁白） |
 
 ### 1.3 核心体验一句话
 
 > "为了保住寨子，我不得不烧掉了一封能让我回家的信。"
 
 玩家在一局 10 分钟的微型自走棋中，反复面对"典当客批换钱救场"还是"死守客批赌翻盘"的残酷抉择。水客是回家的唯一希望，但他脆弱如纸，你必须用肉身为他筑墙。而当你修好了土楼，它真的会反过来庇护你。

 ## 2. 核心 Gameplay 重构
 
 ### 2.1 做减法——删除的系统
 
 | 删除项 | V1.6 设计 | 删除理由 |
 |---|---|---|
 | 利息系统 | 每 10 金产 1 利息（上限 5） | 鼓励"攒钱不打"的消极心流，与 10 分钟极限生存冲突 |
 | 连胜/连败奖励 | 2/3/4 连胜额外 +1/+2/+3 金币 | 连败奖励鼓励"故意输攒钱"，破坏叙事沉浸 |
 | 3 星追求 | 需 9 个同名棋子合 3 星 | 6 回合短局无法支撑 3 星经济循环 |
 | 耕读羁绊 | 农夫+教书先生触发攻速加成 | 简化羁绊层，聚焦宗族 |
 | 后勤无敌位 | 水客/乡贤不上场、不占人口、不死 | 与 v2.0"死保水客"核心张力矛盾 |
 | 农夫产金技能 | 每 2 回合自动产 1 金币 | 被动印钞弱化"典当"抉择的紧迫感 |
 
 ### 2.2 做加法——三大核心微创新
 
 #### 创新 A：典当（经济抉择）
 
 > **情感逻辑**："为了保住寨子，我烧掉了回家的信。"
 
 - **触发**：备战阶段任意时刻，玩家可点击"典当"按钮
 - **代价**：失去 1 封客批（胜利目标倒退一步）
 - **收益**：立即获得 15 金币
 - **限制**：仅当客批 ≥ 1 时可用
 - **战略意义**：当你快撑不住时——是赌一把不典当、靠运气翻盘？还是牺牲一封信用金币买棋子救命？没有正确答案，只有代价。
 
 #### 创新 B：土楼庇护（双线交织）
 
 > **情感逻辑**："我守护的家，也在庇护着我。"
 
 家园修复不再是纯视觉升级，而是直接反哺前线战斗：
 
 | 修复阈值 | 名称 | 视觉解锁 | 战斗 Buff |
 |---|---|---|---|
 | 33% | 修缮·初见 | 水井出水特效 | 全队开场获得最大生命值 20% 护盾 |
 | 66% | 翻新·同心 | 外墙补全特效 | 全队攻速 +15% |
 | 99% | 焕然·不屈 | 祠堂灯火通明特效 | 全体棋子获 1 次免死（致死打击时保留 1 血并无敌 1.5 秒） |
 
 #### 创新 C：死保水客（阵型张力）
 
 > **情感逻辑**："他是回家的唯一希望，但他脆弱如纸。"
 
 - 水客是产出客批的**唯一来源**
 - 水客属性：1 费、0 攻击、中等血量、极度脆弱
 - 敌方新增"刺客"单位（械斗火），AI 行为：开局跳跃至我方后排，优先锁定水客
 - 玩家必须布置守卫在后排贴身保护水客——不再是"前肉后输出"的经典阵型，而是"肉身筑墙，死保核心"

 ## 3. 极简经济学
 
 ### 3.1 收入结构
 
 | 来源 | 数值 | 说明 |
 |---|---|---|
 | 回合工资 | 每回合固定 5 金币 | 无利息、无连胜/连败加成 |
 | 典当客批 | 15 金币 / 封 | 主动触发，扣减胜利进度 |
 | 出售棋子 | 按 cost 退还 | 1 星退 cost，2 星退 cost×3 |
 | 刷新商店 | 1 金币 / 次 | 相比 V1.6 的 2 金币降价，适配低收入 |
 | 升人口 | 4 金币 | 人口上限从 3 升到 6 |
 
 ### 3.2 典当机制详解
 
 ```
 ┌──────────────┐    点击"典当"    ┌──────────────┐
 │  客批 ≥ 1    │ ───────────────→ │  客批 -1      │
 │  金币 = n    │                  │  金币 +15     │
 └──────────────┘                  └──────────────┘
       ↑                                  │
       │        可能透支到无法通关          │
       └──────────────────────────────────┘
               但你不典当这一回合可能就死了
 ```
 
 - 每局最多典当次数 = 拥有的客批数（最多 5 次理论上限）
 - 典当是**不可逆**的——典出去的批，就真的回不来了
 - 设计意图：制造"烧信换钱"的罪恶感，这是本作区别于所有自走棋的核心情感记忆点

 ## 4. 土楼庇护机制
 
 ### 4.1 桑梓值的获取与转化
 
 ```
 水客战场存活且我方胜利
         │
         ▼
    +1 客批 + 20 桑梓值
         │
         ▼
   乡贤在场？── 否 ──→ 桑梓值直接转化为家园修复值（1:1）
         │
         是
         │
         ▼
   转化率 +50%（即 20 桑梓值 → 30 家园修复值）
 ```
 
 ### 4.2 三阶段 Buff 汇总
 
 | 阶段 | 修复值 | 名称 | Buff | 设计意图 |
 |---|---|---|---|---|
 | 一 | 0–32 | 破败 | 无 | 开局状态，危机感 |
 | 二 | 33–65 | 修缮·初见 | 开场 20% 生命护盾 | 中期生存拐点 |
 | 三 | 66–98 | 翻新·同心 | 全队攻速 +15% | 后期输出拐点 |
 | 四 | 99–100 | 焕然·不屈 | 全员免死 1 次 | 终局翻盘希望 |
 
 家园修复值**只升不降**（已修好的不回退），与 V1.6 一致。

 ## 5. 角色与阵型
 
 ### 5.1 战斗棋子（5 种）
 
 | 棋子 | cost | HP | ATK | 攻速 | 护甲 | 射程 | 定位 |
 |---|---|---|---|---|---|---|---|
 | 农夫 | 1 | 450 | 35 | 0.6 | 5 | 近战 | 廉价前排，人海战术 |
 | 围屋守卫 | 2 | 950 | 40 | 0.5 | 25 | 近战 | 肉盾，水客贴身保镖 |
 | 教书先生 | 3 | 550 | 45 | 0.6 | 8 | 中排 | 相邻棋子攻速 +10% |
 | 风水先生 | 4 | 600 | 60 | 0.65 | 10 | 远程 | 主力输出 |
 | 族长 | 5 | 800 | 75 | 0.7 | 15 | 中排 | 全队增益光环 |
 
 ### 5.2 后勤棋子（2 种——上场参战）
 
 | 棋子 | cost | HP | ATK | 攻速 | 护甲 | 射程 | 定位 |
 |---|---|---|---|---|---|---|---|
 | **水客** | 1 | 400 | 0 | 0 | 5 | 近战 | 0 攻击，客批唯一来源，极度脆弱 |
 | **乡贤** | 2 | 600 | 25 | 0.6 | 10 | 远程 | 低攻击，桑梓→修复转化率 +50% |
 
 > **关键变化**：水客与乡贤在 V2.0 中必须上场参战，可以被敌方攻击和击杀。这是"死保水客"阵型张力的前提。
 
 ### 5.3 水客胜负逻辑
 
 | 战斗结果 | 水客状态 | 收益 |
 |---|---|---|
 | 我方胜利 | 水客存活 | +1 客批 + 20 桑梓值 |
 | 我方胜利 | 水客战死 | 不掉寨子血量，但无客批、无桑梓值 |
 | 我方失败 | 任意 | 寨子存续度 -1 |
 
 这把"胜利但不掉血也不得批"设计成了一个情感惩罚——你赢了，但你没有保护好最重要的人，所以这封回家的信，丢了。
 
 ### 5.4 敌方刺客单位
 
 新增敌人类型：**械斗火（刺客）**
 
 - AI 行为：开局第一帧直接跳跃至我方距离最远的后排棋子
 - 优先目标：水客（距离我方阵线最远）
 - 设计意图：强制玩家在后排放置守卫贴身保护水客，形成"前肉→中输出→后守在保人"的三层阵型结构
 
 ### 5.5 敌人池（7 种，含新增刺客）

 |---|---|---|---|---|
 | 迁海碑 | 坦克 | 650 | 35 | 高护甲，前排 |
 | 路引关吏 | 战士 | 500 | 42 | 均衡 |
 | 猪仔契 | 控制 | 720 | 32 | 高血量 |
 | 饿虎山 | 输出 | 580 | 55 | 高攻击 |
 | 红头船 | 远程 | 480 | 50 | 中排输出 |

 ## 6. 一局因果链（胜负判定）
 
 ### 6.1 胜利条件
 
 ```
              开局
               │
    ┌──────────┼──────────┐
    ▼          ▼          ▼
 存续度=2   客批=0    家园修复=0
    │          │          │
    │    ┌─────┴─────┐    │
    │    ▼           ▼    │
    │  水客存活    水客战死 │
    │  +1客批     无客批   │
    │  +20桑梓    无桑梓   │
    │    │           │    │
    └────┼───────────┴────┘
         │
    6 回合结束
         │
    ┌────┴────┐
    ▼         ▼
  存续度>0  存续度≤0
    │         │
  ┌─┴─┐      ▼
  ▼   ▼   风浪抢救结局
 客批  客批   （失败）
 ≥5   <5
  │     │
  ▼     ▼
完美  遗憾
归乡  留守
```
 
 ### 6.2 三种结局
 
 | 结局 | 条件 | 情感调性 |
 |---|---|---|
 | **完美归乡** | 存续度 > 0 且 客批 ≥ 5（第 6 关结束） | 温暖圆满：水客带着侨批登上归船，土楼灯火通明 |
 | **遗憾留守** | 存续度 > 0 但 客批 < 5（第 6 关结束） | 温情遗憾：人活着，寨子保住了，但回家的票没攒够 |
 | **风浪抢救** | 存续度 ≤ 0（中途出局） | 悲壮守护：寨子散了，在风浪中抢救散落的侨批碎片 |
 
 三种结局共用同一个手势接信过场，但信封数量、信件内容、背景氛围不同。

 ## 7. 战斗与数值公式
 
 ### 7.1 伤害公式
 
 $$Actual\\ Damage = Attack \\times \\frac{100}{100 + Armor}$$
 
 - 与 V1.6 一致，沿用已验证公式
 - 护甲边际递减：100 护甲 ≈ 50% 减伤，200 护甲 ≈ 67% 减伤
 
 ### 7.2 升星公式
 
 - 凑齐 **3 个同名 1 星棋子** → 合成为 1 个 2 星棋子
 - 属性计算：$$HP_{2\\star} = HP_{1\\star} \\times 2$$，$$Attack_{2\\star} = Attack_{1\\star} \\times 2$$
 - 攻击速度、护甲、射程不变
 - **不追求 3 星**（6 回合短局无法支撑 3 星经济循环）
 
 ### 7.3 战斗时长
 
 | 参数 | 值 |
 |---|---|
 | 战斗 Tick | 100ms |
 | 最大时长 | 40 秒（400 ticks） |
 | 备战时限 | 30 秒 |
 | 超时判定 | 按双方剩余血量%判定胜负 |
 
 ### 7.4 宗族羁绊
 
 唯一保留羁绊：**客家宗族（hakka）**
 
 | 同族棋子数 | 效果 |
 |---|---|
 | 2 | 全队攻击 +10% |
 | 3 | 全队攻击 +20% |
 | 4 | 全队攻击 +30% |
 
 ### 7.5 关卡难度曲线
 
 | 关卡 | 敌人数 | 难度系数 | 出现敌人类型 |
 |---|---|---|---|
 | 1–2 | 3 | ×1.0 | 迁海碑、路引关吏 |
 | 3–4 | 4 | ×1.5 | 猪仔契、饿虎山、红头船 |
 | 5–6 | 5 | ×2.0 | 全部 + **械斗火（刺客）**必出 |
 
 械斗火从第 5 关开始出现，作为终局强度验证——玩家必须已经建立"死保水客"的阵型。

 ## 8. UI 与视觉反馈（Juice）
 
 作为黑客松项目，打击感和情绪反馈比复杂的系统更重要。
 
 ### 8.1 典当的痛感
 
 | 要素 | 设计 |
 |---|---|
 | 触发 | 点击"典当"按钮 |
 | 视效 | 信封燃烧/撕裂动画（火焰从边角吞噬信纸），随后金币从灰烬中哗啦啦掉落 |
 | 音效 | 沉重的一声"咚"（类似印章/心沉），接着金币清脆的叮当声 |
 | 情感目标 | 让玩家感到"用血换钱"的罪恶感——钱来了，但代价是心头肉 |
 
 ### 8.2 土楼反哺的爽感
 
 | 阈值 | 视效 | 音效 |
 |---|---|---|
 | 33% | 全屏金色粒子向棋盘中央聚拢，水井涌水动画 | 泉水叮咚 → 短促的管弦上扬 |
 | 66% | 外墙修复光效从下往上扫过，棋子身上出现金色护盾特效 | 墙体砌筑音效 → 恢弘的宗族鼓点 |
 | 99% | 祠堂灯火逐盏亮起，棋盘边缘泛起暖色光晕，"免死"Buff 棋子身上出现守护光环 | 灯笼点燃音 → 悠长的客家山歌片段 |
 
 ### 8.3 水客的危机
 
 | 状态 | 视觉 | 听觉 |
 |---|---|---|
 | 血量 > 50% | 正常 | 正常 |
 | 血量 30%–50% | 屏幕边缘微微泛黄 | 轻微急促呼吸音 |
 | 血量 < 30% | 屏幕边缘泛红 + 脉冲暗角 | 心跳声渐强，背景音乐压低声场 |
 | 水客死亡 | 屏幕瞬间灰白 0.5 秒 | 沉重的坠落音 + 客家话叹息 |
 
 ### 8.4 备战阶段视觉引导
 
 - 当选中棋子准备布阵时，推荐站位高亮（绿色 = 适合攻击位，蓝色 = 保护水客的安全位）
 - 当敌方阵容有刺客（第 5–6 关）时，水客身边出现红色警示圈，提示"此处需要保护"

 ## 9. AI 落点
 
 ### 9.1 主落点：机制叙事生成（Ludonarrative Generation）
 
 本版本的 AI 策略聚焦一个低成本、高情感回报的落点——**AI 结算旁白**。
 
 #### 触发时机
 
 每次战斗结算后（无论胜负），系统收集本回合的关键事件，调用 AI 接口生成一句客家侨批风格的动态旁白。
 
 #### 输入上下文
 
 ```ts
 type TurnNarrativeInput = {
   turn: number;
   // 关键事件标志
   events: {
     didPawn: boolean;          // 是否典当了客批
     pawnCount: number;         // 典当了几封
     waterGuestDied: boolean;   // 水客是否战死
     waterGuestSurvived: boolean;
     won: boolean;
     clutchUnit?: string;       // 残血翻盘的单位名（如"农夫"）
     homeRepairMilestone?: number; // 是否触发了 33/66/99 阈值
   };
   currentKebi: number;
   currentHomeRepair: number;
   survival: number;
 };
 ```
 
 #### 输出格式
 
 ```ts
 type TurnNarrative = {
   text: string;    // 一句话旁白，客家侨批语气
   author: string;  // 虚拟署名（如"阿发 叩上"）
 };
 ```
 
 #### Prompt 示例（系统侧）
 
 ```
 你是一个在海外谋生的客家人，用侨批的语气给家乡亲人写一句话。
 语气要求：简朴、克制、深情，像真正一百年前的客家侨批。
 长度要求：10–25 个汉字。
 
 本回合事件：
 - 我{[典当了]} {1} 封好不容易攒下的侨批，换了盘缠。
 - 水客在战斗中{[活下来了]}。
 - 我们{[赢了]}这一仗。
 - 家园修复到了 {33%}，水井终于又出水了。
 
 请生成一句符合上述语气的旁白。
 ```
 
 #### 示例输出
 
 | 场景 | AI 生成旁白示例 |
 |---|---|
 | 典当了客批 | "阿嬷，为了保住大门，我把二弟寄来的信当了换了些盘缠，我对不住他……" |
 | 水客战死但赢了 | "寨子守住了，可信丢了。阿发那封信，怕是要沉在海底了。" |
 | 残血翻盘 | "就剩一个农夫，拄着锄头也把仗打赢了。祖先保佑。" |
 | 土楼 33% 解锁 | "井里又冒水了。阿妈，是不是你在老家也在打水？" |
 | 土楼 99% 解锁 | "祠堂的灯亮了。我好像听见你在叫我回家。" |
 
 #### 技术实现要点
 
 - **极低成本**：一次轻量 Prompt 请求，约 50–100 tokens 输入，20–30 tokens 输出
 - **降级友好**：AI 不可用时使用本地 fallback 文案池（12–15 条预设，按事件类型分组随机选取）
 - **不需要流式**：一句话旁白，直接返回不需要打字机效果
 - **缓存策略**：相同事件组合可复用生成结果，降低 API 调用次数
 
 ### 9.2 辅助落点：AI 对手难度（P1，依工期取舍）
 
 - 第 5–6 关的敌方阵容由 AI 根据玩家当前阵容动态生成（而非固定配置）
 - 但考虑到 4 天开发周期的风险，P0 先用固定配置 + 关卡难度系数实现，AI 对手留作 P1

 ## 12. 实施建议
 
 ### 12.1 迁移优先级
 
 | 优先级 | 改动 | 工作量估算 |
 |---|---|---|
 | P0 | 删除利息/连胜经济代码 | 小（纯删除） |
 | P0 | 新增典当机制（engine + UI） | 中（新 reducer action + 按钮 + 视效） |
 | P0 | 水客/乡贤改为上场棋子 | 中（改 Support 概念为 Piece） |
 | P0 | 新增刺客 AI（械斗火跳后排） | 中（battle 模块新增跳跃逻辑） |
 | P0 | 土楼 Buff 系统（三阶段阈值触发） | 中（progression 模块新增 buff 发放） |
 | P0 | AI 结算旁白（API + fallback） | 小（纯 API 调用 + 本地文案池） |
 | P1 | 典当/土楼 Juice 视效 | 大（粒子、动画、音效） |
 | P1 | 水客危机 UI 反馈（红屏/心跳） | 中 |
 | P1 | AI 对手难度 | 大（需调优） |
 
 ### 12.2 风险提示
 
 - **典当可能被滥用**：如果玩家典当太多次导致客批 < 5 无法通关，会产生挫败感。需要在大约第 3–4 关时通过 UI 提示"你的客批可能不够回家了"。
 p1-预计后期设计填补
 - **水客存活率**：如果水客在前几关频繁阵亡导致玩家始终攒不到客批，会产生无力感。考虑给水客初始一个 1 次免死保护（类似土楼 99% 的免死效果，但仅限第 1–2 关）。
 p0-明日todolist
 - **土楼 Buff 叠加**：土楼三阶段的 Buff 是累积的（33% 护盾 + 66% 攻速 + 99% 免死），需要在数值上验证不会导致终局过于碾压。
p2-数值验证板块
