# 《客批》UI 设计规范 v1

> 目的：把 UI 改版方向定成可执行的视觉规范，解决当前「后台面板感」与美术文档脱节的问题
> 依据：[美术风格设定 v1](kepi_art-style-design_v1.md) · [素材与媒体计划 v1](kepi_assets-and-media-plan_v1.md)
> 更新：2026-06-26 — 沉浸式 HUD + 木框纸纹 + 暖色黄昏，面向 16:9 路演横屏；同步对齐图标徽章化与素材计划确认稿

## 1. 设计目标

### 1.1 首要目标

**Demo 观感第一**：评委第一眼应感到「这是一款完整游戏」，而不是 shadcn 调试页。

### 1.2 已确认方向（2026-06-26 对齐）

| 维度 | 定稿 |
|---|---|
| 布局 | 沉浸式全屏：Canvas 占主视觉，React 只做 HUD / 弹窗 |
| 装饰语言 | 木框、宣纸、侨批信纸质感 |
| 场景参考 | 《大鱼海棠》：诗意、暖色、留白 |
| 色调 | 暖色黄昏：木褐底 + 琥珀光 + 靛蓝点缀 |
| 信息密度 | 稀疏 HUD：仅核心数值常驻，其余按需展开 |
| Canvas / UI 分工 | Canvas 负责氛围 + 棋盘 + 角色；React 负责 HUD 与交互层 |
| 字体 | 全 sans（Noto Sans SC），保证 16:9 投影可读 |
| 纹理素材 | 现阶段 **CSS 模拟木框纸纹**（`kepi-paper` / `kepi-wood-frame`）；PNG 到位后可替换 |
| 图标语言 | 圆徽章 / 轻徽章边框 + 中心符号，统一 PNG 透明底 |

### 1.3 路演屏幕

- **主目标**：16:9 横屏投影（1920×1080 或 1280×720）
- **安全区**：核心 HUD 与按钮落在中央 90% 宽、上下各留 5% 边距，避免投影裁切
- **次要兼容**：同布局在 1366×768 笔记本可玩，但不单独做竖屏适配

## 2. 当前问题诊断

以下问题需**整体重做**，不是局部微调：

| 问题 | 现状 | 目标 |
|---|---|---|
| 布局气质 | 左 Canvas + 右 shadcn Card 栏，像 SaaS 后台 | 全屏 Canvas + 浮动 HUD |
| 色彩体系 | `--background` 仍为默认白/灰；场景 token 偏冷 | 统一暖木褐 / 琥珀 / 靛蓝 token |
| 组件语言 | Card、outline Button、Badge 通用组件直出 | 木框面板、纸签标签、木牌按钮 |
| 信息架构 | Header 与 StatsPanel 重复展示 | 稀疏常驻 + 信笺抽屉 |
| Canvas 与 UI 割裂 | 画布在做土楼，UI 是 modern flat | 共享木框 / 纸纹 / 暖光语言 |
| 占位感 | 棋子色块圆、商店 grid 按钮 | 立绘或风格化棋子 + 货摊木牌商店 |
| 调试感外露 | 顶栏「调试 / 新局 / Kepi · 对局主流程」占主视觉 | 收入设置菜单，主界面无开发痕迹 |

## 3. 视觉语言

### 3.1 色彩 Token（CSS Variables）

在 `globals.css` 中逐步替换 shadcn 默认中性色，以 `--kepi-*` 为主、`--background` 等 shadcn token 映射到同一体系。

| Token | 用途 | 建议值（方向） |
|---|---|---|
| `--kepi-scene` | 全屏背景基色 | 深木褐 `#3d2e1f` ~ `#4a3728` |
| `--kepi-scene-glow` | 黄昏光晕 overlay | 琥珀 `#e8a849` @ 12–18% opacity |
| `--kepi-panel` | HUD / 弹窗纸面 | 米宣 `#f5ead6` @ 88–92% |
| `--kepi-panel-border` | 木框描边 | 深褐 `#5c4033` |
| `--kepi-ink` | 主文字 | 墨褐 `#2c2416` |
| `--kepi-ink-muted` | 次要文字 | `#6b5b4f` |
| `--kepi-accent` | 靛蓝点缀（我军 / 高亮） | `#4a6fa5` |
| `--kepi-accent-soft` | 靛蓝浅底 | `#8ecae6` @ 22% |
| `--kepi-danger` | 存续 / 失败 | `#7b4b4b` |
| `--kepi-enemy` | 敌军强调 | `#c1121f` |
| `--kepi-gold` | 金币 | `#d4a574` |
| `--kepi-tulou-glow-*` | 土楼三阶段 | 沿用现有 ruined / repairing / renewed |

**原则**：不用纯白 `#fff` 作面板底；不用 shadcn 默认灰 Card；敌我色遵循美术文档（我军靛蓝温润，敌军青灰朱红）。

### 3.2 纹理与材质

现阶段使用 CSS 工具类（见 `globals.css`：`kepi-paper`、`kepi-wood-frame`、`kepi-wood-btn` 等）模拟木框与宣纸。PNG 纹理（§2.3A）到位后替换背景，不改组件结构。

- **木框**：HUD 外框、主按钮、商店条容器
- **宣纸**：面板内容区、战况抽屉、Tooltip 底
- **信纸边**：弹窗、侨批相关、结局层
- **禁止**：纯 shadcn 默认白 Card 作为终稿

### 3.3 字体

- **唯一正文字体**：Noto Sans SC（已在 `layout.tsx`）
- **字重**：标签 400、数值 500、标题 700
- **字号（16:9 基准）**：
  - HUD 数值：18–20px
  - HUD 标签：12px
  - 弹窗正文：14–16px
  - 最小可读：12px（投影环境下不再缩小）
- **不做**：标题宋体 / 书法字（路演可读性优先；Logo 可后续单独讨论）

### 3.4 圆角与间距

- 不用大圆角 Card 美学；外轮廓由**木框纹理**决定
- 元素间距：8px 网格；HUD 内 padding 12–16px
- 浮动层阴影：暖色 soft shadow `0 4px 24px rgba(44, 36, 22, 0.35)`，避免冷灰 shadow

## 4. 布局结构

### 4.1 主对局页（prep / battle / settlement）

```
┌──────────────────────────────────────────────────────────────┐
│ ░░░░░░░░░░░░░ 顶部 HUD 条（木框纸签，高 56–64px） ░░░░░░░░░░░ │
│  [客批 2/5]  [金币 10]  [第 1/6 关]  [备战]     [信笺☰] [⚙] │
├──────────────────────────────────────────────────────────────┤
│                                                              │
│                    Canvas 全屏（absolute inset-0）             │
│              土楼背景 + 棋盘 + 棋子 + 战斗特效                  │
│                   + 暖色 vignette overlay                    │
│                                                              │
├──────────────────────────────────────────────────────────────┤
│ ░░░ prep 专用：底部商店木条（高 96–120px，可收起） ░░░░░░░░░░░ │
│   [棋1][棋2][棋3][棋4][棋5]   [刷新] [升人口]      [开战 ▸]   │
└──────────────────────────────────────────────────────────────┘
```

- **Canvas**：`position: absolute; inset: 0; z-index: 0`
- **HUD / 商店条**：`z-index: 10+`，pointer-events 按需开启
- **移除**：右侧 320px 固定侧栏

### 4.2 信息分层

#### 常驻 HUD（稀疏）

| 项 | 说明 |
|---|---|
| 客批 | `kebi / kebiThreshold`，达阈值高亮 |
| 金币 | 当前 gold |
| 关卡 | `stage / totalStages` |
| 阶段 | 备战 / 战斗 / 结算（短标签） |

#### 按需展开（信笺抽屉 / 弹窗）

- 存续度、桑梓值、家园修复%、连胜连败
- 水客 / 乡贤状态
- 替补席棋子列表（bench）

#### 阶段专属

| 阶段 | UI |
|---|---|
| prep | 底部商店木条 + 开战按钮 + 替补席入口 |
| battle | 顶部 HUD 即可；可选极简血条在 Canvas 内 |
| settlement | 全屏胜利结算过场（遮住棋盘与棋子）→ 中央纸面结算卡 + 「继续」；失败为短反馈 + 摘要卡 |
| ending | 全屏结局层（独立规范，见 Phase 5） |

#### 4.2A 胜利结算过场 UI

- 胜利结算阶段先进入全屏 cinematic overlay，背景必须不透明或近似不透明，**不露出己方棋子、敌人、棋盘格**。
- 过场分镜：海上送信 → 土楼门前交信 → 桑梓显现 → 乡贤修楼 → 土楼阶段变化 → 摘要卡。
- 过场必须有「跳过」按钮；跳过后直接进入摘要卡，不影响数值。
- 摘要卡只承担"账本"功能：客批 +1、桑梓 +1 已用于修缮、家园修复 +N%、继续下一关。
- 普通胜利过场约 6 秒；跨 34% / 67% 的胜利保留完整升级镜头；失败结算不使用完整胜利过场。
- 技术建议：DOM 分层动画为主，背景图 pan/zoom，土楼三阶段图复用 Canvas / 静态资源。

### 4.3 设置与调试

- **设置 ⚙**：音量、手势、新局、续局
- **调试页**：仅 `/debug` 路由，主界面不出现「调试」按钮与「Kepi · 对局主流程」文案

## 5. 组件规范

### 5.1 组件映射（旧 → 新）

| 现有 | 目标 | 纹理依赖 |
|---|---|---|
| `GameHeader` + `StatsPanel` | 合并为 `HudBar` + `LetterDrawer` | `kepi_ui_hud-tag_*`, `kepi_ui_paper-cream_*` |
| `ShopPanel` | `ShopStrip` 底部横条 | `kepi_ui_frame-wood_*`, `kepi_ui_shop-slot_*` |
| `Card` + `Button` | `WoodPanel`, `WoodButton` | 木框 + 木按钮 9-slice |
| `BenchPanel` | 抽屉内或 Canvas 热区 | 纸面列表 |
| `ToastHost` | 纸签 toast，右上角 | `kepi_ui_hud-tag_*` |
| `GameDialogs` | 信纸弹窗 | `kepi_ui_paper-letter-edge_*` |

### 5.2 按钮状态

| 状态 | 视觉 |
|---|---|
| default | 木按钮 normal 纹理 |
| hover | 亮度 +4%，可选轻微 scale 1.02 |
| disabled | 木按钮 disabled 纹理，文字 `--kepi-ink-muted` |
| primary（开战） | 同木按钮 + 琥珀内发光或靛蓝描边 |

### 5.3 图标

使用 [素材计划 §2.3](kepi_assets-and-media-plan_v1.md#23-ui-与图标) 统一风格图标，尺寸 24×24（HUD）/ 32×32（商店），PNG 透明底，圆徽章 / 轻徽章边框 + 中心符号，扁平但带一点体积感，主色为靛蓝 / 木色 / 琥珀金。

## 6. Canvas 协同

### 6.1 Canvas 职责

- 土楼三阶段背景与暖色天光
- 圆形 / 宽画幅棋盘
- 棋子、敌人、战斗特效
- 可选：Canvas 内绘制极简数值（如棋子脚下星级）

### 6.2 React 职责

- 顶部 HUD、底部商店条
- 弹窗、抽屉、Toast
- 点击热区（若棋子选取不在 Canvas 内完成）

### 6.3 统一氛围层

Canvas 上层可叠 `--kepi-scene-glow` 与 `kepi_ui_vignette-warm.png`（全屏 multiply/overlay），使 HUD 与场景同处黄昏光场。

## 7. 实施顺序（Phase 4 预览）

纹理 P0 到位前**不开始** UI 终稿开发；到位后按序：

1. **Token**：`globals.css` 暖色体系，移除默认 shadcn 白底
2. **布局**：`GameShell` 改为全屏 Canvas + 浮动 HUD，删除右侧栏
3. **基础组件**：`WoodPanel` / `WoodButton` / `HudBar`
4. **商店条**：`ShopStrip` 替换 `ShopPanel`
5. **信笺抽屉**：合并战况信息，删除重复 Header
6. **Canvas 氛围**：vignette + 土楼阶段光晕
7. **收尾**：隐藏调试入口、Toast/Dialog 换肤

## 8. 验收标准（Demo）

- [ ] 16:9 投影下 3 米外可辨客批 / 金币 / 开战
- [ ] 主界面无 shadcn 默认白 Card、无「调试」字样
- [ ] 所有可见面板均使用木框 / 纸纹素材，非纯 CSS 边框
- [ ] Canvas 占 ≥ 85% 视口高度
- [ ] 与 [美术风格设定 v1](kepi_art-style-design_v1.md) 暖木靛蓝一致，无冷灰 SaaS 感
- [ ] 断网本地加载全部 UI 纹理（`public/images/ui/`）
